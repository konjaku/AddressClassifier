# 地址/POI 名称分类器 · 重做设计(通名驱动确定性分类)

- 日期:2026-06-15
- 状态:已通过头脑风暴评审,待写实现计划
- 范围:本地批处理版分类器的完整重做(`/Users/mac/Code/Work/AddressClassifier/`)

## 1. 背景与目标

把 POI 的「名称/地址」批量映射到 `categories.xlsx` 的标准分类(16 一级 / 52 二级 / 326 三级)。

- 原系统由 OpenAI Codex 开发,实测正确率约 80%,不够。
- 已被集成进一台**非公网(内网)服务器**,该服务器代码/文件**暂时不能改**,只能看到导出与错误;其导出格式与本地 `data/output/` 差别较大,且像是更早、更依赖模糊检索的版本。
- 本设计是**新的本地实现**,将来条件允许时再替换/回灌到内网。

**目标正确率:~90%(用户确认 90% 即够)。**

## 2. 硬约束

1. **完全离线**:最终运行在内网,**不能调用任何外部 API**。
2. **纯传统算法**:不使用大模型,也不使用 embedding 模型。
3. **没有固定真值集**:只能抽样人工核对 → 系统必须**极度可解释**,并能从抽查纠错中**反哺词典、沉淀回归集**。
4. 批处理形态:Excel 进、Excel 出(沿用现状)。

## 3. 为什么重做(根因,已复现并量化)

对「无强规则覆盖」的一大类名字(政府厅/局/委/办、事业单位中心/院/站、组合式机构名),旧系统退回到**「字符 ngram 模糊相似度撞类名」**,于是按无关共享语素乱配:乡村振兴**局**→度假**村**、X**中心**→考试**中心**、管**委会**→**村委会**。中文机构名是组合式的,语义中心在**通名(中心词/后缀)**,模糊相似度根本无法理解机构类型。

补充确认的实现层缺陷(供参考,不在本设计内修补,而是整体替换):
- `text_rules.py` 的 `_rule_matches` 把 `positive_any` 与 `positive_regex_any` 当成 AND,误杀 `medical_tcm_rehab`(国医馆/中医馆等)。
- `retrieval.py` 的 `decide` 中 `risk_keyword_missing` 降级会**静默丢弃已正确命中的强规则**(如皮肤病专科→专科医院被降级成"其他服务")。

量化(625 条样本):22% 落入「其他服务」兜底;字符检索 63% 的行 top1 是「省会名/地级市地名/火车站」等地理噪声类。同时验证:**92.3% 的名字带可识别通名**,残余多数仍可用更全词典覆盖 → 纯传统现实可达 ~88–93%。

## 4. 方案选型

选 **A:通名驱动的确定性分类器**(已评审通过)。

- B(通名路由 + 类内经典 ML):需要没有的标注语料,可解释性差 → 否决。
- C(在旧架构上修 bug + 补规则):保留模糊相似度坏底盘,现实只到中 80% → 否决。

A 与「离线 + 纯传统 + 抽查迭代」的约束最契合:完全可解释,错因可定位,词典可从单条纠错增量生长。

## 5. 架构:管线九步

```
Excel(名称/地址)
  ① 规范化与切分    全/半角、去噪、名称vs地址、纯地址识别
  ② 实体定位        剥离行政前缀(湖南省/长沙市/区)与登记/分店后缀((有限公司)/(XX店)),定位主体
  ③ 通名抽取        从主体抽中心词,取最右/最长为主通名;可多个
  ④ 通名→候选类     映射到候选大类(可跨一/二/三级);词典从"适用对象"自动生成
  ⑤ 修饰词消歧      候选>1时用修饰词定夺(中心:考试/应急/会展;医院:宠物/专科/综合)
  ⑥ 行政级别判定    政府机关按前缀定级(国家/省/市/区)
  ⑦ 优先级裁决      功能实体>登记主体>地址结构;排斥规则(地址里"银行对面"不算银行)
  ⑧ 品牌兜底        无通名时查品牌词典;仍无→标"待复核",绝不模糊乱撞
  ⑨ 置信分层        gold(通名清晰且无歧义)/silver(消歧不定)/review(无通名/未知)
Excel(最终类 + 命中通名 + 候选 + 消歧依据 + 级别 + 置信 + 复核标记)
```

### 四条核心原则

- **层级感知**:输出能确信的**最深一层**;拿不准三级就退回二级(如银行网点→「金融服务」二级),绝不为凑三级而错判,也绝不塞「其他服务」。
- **零模糊相似度**:彻底删除字符 ngram 撞类名;无法判定显式标「待复核」。
- **可解释**:每行带「凭哪个通名、被哪条修饰词消歧、判到几级」。
- **陷阱通名硬约束**:裸「学校」禁止→「学校报名处」;多义通名(医院/中心/院/店/馆/场/站/园 等)**必须**过消歧,不允许默认取首个候选。

## 6. 分类体系的利用

- **层级树**:16 一级 → 52 二级 → 326 三级,提供天然的"粗路由→细化"。部分业态只在二级落点(如「金融服务」),系统必须支持按层级输出。
- **描述列即词典**:每个类的「向量匹配分类描述」含 `适用对象:餐馆、饭店、酒楼…` 与 `层级路径:餐饮 > 餐馆 > 中餐馆`。**基础通名/同义词词典直接从该列自动抽取**,大幅降低手写与维护成本。
- 已确认的多义通名歧义度(命中三级类数):店 29、场 27、馆 19、站 16、院 13、社/会/所/园 6–9、中心 7、医院 4 —— 这些是消歧模块的重点。
- 已确认的陷阱:三级里含「学校」者只有「学校报名处」;含「银行」者只有「ATM/自助银行」(真正银行网点→二级「金融服务」)。

## 7. 代码模块(小而单一,各自可独立测试)

| 模块 | 职责 | 管线步 |
|---|---|---|
| `normalize.py` | 文本规范化、名称/地址切分、纯地址识别 | ① |
| `segment.py` | 剥离行政前缀 + 登记/分店后缀,定位主体 span | ② |
| `tongming.py` | 通名抽取(最右/最长)+ 映射候选类,加载词典 | ③④ |
| `disambiguate.py` | 多义通名修饰词消歧 + 政府行政级别判定 | ⑤⑥ |
| `gazetteer.py` | 品牌/强实体词典兜底 | ⑧ |
| `arbitrate.py` | 优先级裁决 + 排斥规则 + 置信分层 | ⑦⑨ |
| `classify.py` | 编排 + Excel 读写 + 解释列输出 | 全流程 |
| `build_dict.py` | 离线工具:解析 categories.xlsx 描述列生成基础词典 | 构建期 |

## 8. 词典文件结构(自动生成 + 人工覆盖)

```
dict/
  tongming.auto.yaml      ← build_dict.py 从描述列「适用对象/层级路径」自动生成
  tongming.override.yaml  ← 手工:陷阱通名、补漏、优先级提示
  modifiers.yaml          ← 多义通名消歧规则
  levels.yaml             ← 行政级别前缀规则(国家/省/市/区)
  gazetteer.yaml          ← 品牌强实体
  denylist.yaml           ← 排斥/降权
```

加载时 `override` 覆盖 `auto`。`categories.xlsx` 一变,重跑 `build_dict.py` 即可;人工只维护 override 系列小文件。

## 9. 无真值集下的评测/迭代工作流

```
跑一版 → 按置信分层:
   gold           → 直接采用
   silver/review  → 进「复核队列」(量小,只看这部分)
        ↓ 人工抽查纠错
   每条纠正 → 追加进 eval/regression.jsonl(名称→正确类)
        ↓ 越攒越多 → 成为事实上的评测集
   run_eval.py:对回归集算准确率 + 分类目分布 + 与上一版 diff(谁被改坏)
```

- **未知通名日志**:凡因「无通名」进复核的,记录其候选中心词 → 直接指出下一步该往词典加哪些词(数据驱动地长词典)。
- 回归集的作用:加新词典条目时,保证**已通过样本不回退**。

## 10. 测试策略(实现期 TDD,先写测试)

- **单元测试**:通名抽取(最右/最长)、前后缀剥离、每个消歧组(中心/医院/院/店…)、行政级别、优先级裁决、排斥规则。
- **回归测试**:跑 `eval/regression.jsonl`,断言准确率 ≥ 阈值,且已通过样本不回退。
